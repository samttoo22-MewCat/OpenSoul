🚀 应用修复的步骤

Step 1: 停止现有容器
  $ docker-compose down

Step 2: 删除旧镜像（可选，但推荐）
  $ docker image rm opensoul-api  (如果存在)

  或全量清理：
  $ docker system prune -a

Step 3: 重建镜像
  $ docker-compose build --no-cache

Step 4: 启动容器
  $ docker-compose up -d

Step 5: 验证所有服务
  $ docker-compose ps

  应该显示:
    - opensoul-falkordb    (healthy)
    - opensoul-falkor-browser
    - opensoul-api         (running)

Step 6: 验证 skills 可用性

  查看 Gmail skill:
    $ docker exec opensoul-api ls -la /app/openclaw/skills/gmail/

    输出应该包含:
      SKILL.md
      scripts/
      └── gmail_controller.py

  查看所有 skills:
    $ docker exec opensoul-api ls /app/openclaw/skills/ | wc -l

    输出应该是: 54

Step 7: 检查 API 日志
  $ docker logs opensoul-api

  应该看到:
    Uvicorn running on http://0.0.0.0:6781
    (无错误信息)

Step 8: 测试 API 连接

⚠️ 注意事项
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Docker 镜像大小会增加 (多了 openclaw 目录)
   估计增量: 50-100MB (取决于 node_modules 和缓存)

2. 首次 build 会耗时较长
   预计时间: 3-5 分钟（取决于网络和 CPU）

3. Workspace volume 挂载
   - credentials.json 和 token.json 必须存在于本地
   - 否则 Gmail skill 会在容器启动时失败
   - 但其他 53 个 skills 仍可正常使用

4. 编码配置
   - 已在 docker-compose.yml 中设置 PYTHONIOENCODING=utf-8
   - 这解决了 Windows cp950 编码问题

5. 开发模式
   - volumes 中的 `./openclaw:/app/openclaw` 允许实时修改
   - 不需要重建镜像就能更新 skills（适合开发）
   - 生产环境可移除此 volume
